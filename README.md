# 🎵 Yandex Music Player (React + Vite)

Красивый, плавный и производительный веб-плеер для Яндекс Музыки с интеграцией поиска по YouTube, поддержкой Яндекс.Радио («Моя волна») и продвинутыми визуальными эффектами в стиле Glassmorphism.

---

## ✨ Основные возможности

* **Полная интеграция с Яндекс Музыкой**: воспроизведение треков, поддержка плейлистов, лайков, чартов и новинок.
* **Бесконечный поток «Моя волна»**: умный подбор рекомендаций от Яндекса с возможностью тонкой настройки характера музыки (популярное, любимое, незнакомое), языка треков и настроения.
* **Умный поиск**: находит и проигрывает треки напрямую из Яндекс Музыки, а при их отсутствии автоматически ищет видео/аудио на YouTube через сеть приватных прокси-инстансов Piped.
* **Интерактивные тексты песен (Lyrics)**: динамическая синхронизация слов песни с текущим временем воспроизведения и плавным автоскроллом.
* **Три режима отображения (Layouts)**:
  * *Классический десктопный вид* с боковой панелью управления.
  * *Раздельный экран (Split Screen)* с красивой панелью текста песни справа.
  * *Полноэкранный кинематографический плеер* с размытыми обложками и плавающим текстом на весь экран.
* **Адаптивные визуальные эффекты**: плавные микро-анимации, анимированные 3D-логотипы и динамический фоновый свет (glow), меняющий цвет под обложку текущего трека.

## 🔒 Безопасность и Конфиденциальность

Мы ценим безопасность ваших данных, поэтому плеер спроектирован так, чтобы полностью исключить риск компрометации аккаунта:
* **Авторизация без пароля**: Вход выполняется по официальному протоколу **Yandex Device Flow OAuth**. Вы получаете временный код авторизации в приложении и вводите его на официальной защищенной странице Яндекса (`yandex.ru/activate`). Приложение никогда не видит и не запрашивает ваш пароль.
* **Локальное хранение**: Все полученные токены доступа сохраняются **исключительно на вашем устройстве** в `localStorage` вашего браузера. Они не передаются и не хранятся на сторонних серверах.
* **Прозрачный прокси-сервер**: Встроенный сервер используется только как транзитный ретранслятор для обхода ограничений CORS браузера. Он не собирает логи, не хранит ключи и работает по схеме Stateless.
* **Открытый код**: В исходном коде отсутствуют зашитые приватные ключи или токены — проект полностью безопасен для публикации в открытом доступе на GitHub.

## ⚡ Технологии и Оптимизация производительности

Для достижения максимальной плавности (60+ FPS) на компьютерах любой мощности были применены профессиональные приемы оптимизации:
* **Аппаратное GPU-ускорение**: Тяжелые интерфейсные слои с размытием (`backdrop-filter`) и сложные анимации (цветок Premiere, глаз Dejavu, звезда Daily) вынесены в отдельные слои визуализации видеокарты с помощью `will-change` и 3D-трансформаций (`translate3d`). Это разгрузило центральный процессор и убрало микрофризы при скроллинге.
* **Оптимизация фоновой вкладки (CPU Idle)**: Приложение отслеживает видимость страницы (Visibility API). Если вкладка сворачивается или уходит на задний план, частота циклов обновления прогресс-бара снижается со 120 Гц до экономичных 1 Гц (раз в секунду), снижая потребление ресурсов процессора до минимума.
* **Прямые манипуляции с DOM (Uncontrolled Inputs)**: Обновление временных меток и ползунков перемотки происходит напрямую в обход циклов перерисовки виртуального DOM React через планировщик `requestAnimationFrame`. Это позволило избавиться от задержек интерфейса при ручной перемотке треков.
* **Нативный прокси-слой на Node.js/curl**: Использование системных утилит `curl.exe` на сервере для обхода специфических ограничений по TLS/SSL со стороны Yandex API, обеспечивая стабильный и бесперебойный стриминг.

## 🛠️ Запуск локально

1. Установите зависимости проекта:
   ```bash
   npm install
   ```
2. Запустите сервер разработки:
   ```bash
   npm run dev
   ```
3. Откройте адрес `http://localhost:5173` в браузере.

## 🚀 Деплой (развертывание)

Проект оптимизирован для работы на облачной платформе **Vercel** или **Netlify**:
* При импорте репозитория в Vercel все настройки сборки (Vite) определятся автоматически.
* Прокси-запросы к Yandex API будут автоматически обрабатываться встроенными бессерверными функциями (Serverless Functions) согласно конфигурации `vite.config.js`.
